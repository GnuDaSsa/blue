## [2026-06-12] Chrome 플러그인 연결 점검
- 요청/목표: Computer Use 오류 이후 영상 업무 UI 조작을 Chrome 기준으로 다시 시도한다.
- 점검:
  - Chrome 설치 및 실행 확인: 정상.
  - Codex Chrome Extension 설치/활성화 확인: `Profile 2`에서 정상.
  - Native messaging host manifest 파일은 존재하지만 Windows 레지스트리 키가 없어 Chrome 확장 통신이 실패한다.
- 결정:
  - 현재 상태에서는 Chrome 자동 조작으로 Suno/Runway 실습을 진행할 수 없다.
  - Codex plugin UI에서 Chrome plugin을 재설치하여 native host 등록을 복구해야 한다.
  - 영상 업무 UI 조작은 Chrome 기준으로 하되, 연결 복구 후 Suno/Runway 실습을 재개한다.
- 후속 확인:
  - Codex의 공식 Chrome plugin 설치 요청은 완료됐지만, native host 레지스트리 키는 여전히 생성되지 않았다.
  - Chrome 확장 자체는 설치/활성화되어 있으므로, 남은 문제는 Codex 데스크톱 앱의 native host 등록 복구다.
  - Claude 점검 결과, 매니페스트의 allowed origin과 실제 확장 ID 및 host 경로는 일치하고, native host presence 기록도 있었다.
  - 추가 가설: 확장이 `Profile 2`에만 설치되어 있으므로 Codex/Chrome 자동화가 `Default` 또는 `Profile 1`로 뜨면 연결이 실패할 수 있다.
  - 빠른 복구 절차: Chrome 완전 종료 → `Profile 2`로 Chrome 실행 → `chrome://extensions`에서 Codex 확장 활성화 확인 → Codex에서 Chrome 연결 재시도 → `extension-host` 프로세스 확인.
  - Chrome 업데이트 후 `extension-host` 프로세스가 실행되는 것을 확인했다.
  - 남은 차단점은 Codex 앱의 "컴퓨터 유즈 기능 활성화" 권한 게이트로 보인다. 이 권한이 켜진 뒤 Chrome 연결을 재시도한다.

- 요청/목표: 다른 세션에서 성공한 `[@컴퓨터](plugin://computer-use@openai-bundled)` 흐름을 확인하고, 앞으로 사용자가 명시 태그를 쓰지 않아도 Computer Use 의도로 해석하도록 한다.
- 확인:
  - 최근 세션 `수노 켜고 작업 시작`은 `[@컴퓨터](plugin://computer-use@openai-bundled)`로 시작했고, 별도 작업 디렉터리 `plugin-computer-use-openai-bundled`에서 Computer Use 스킬이 트리거되었다.
  - 해당 세션도 Computer Use 직접 초기화는 `@oai/sky` package exports 오류를 만났고, 이후 Chrome 경로로 전환했다.
  - 같은 세션의 후속 점검에서는 Chrome 실행, Profile 2 확장 설치/활성화, native host manifest/registry가 정상으로 확인되었다.
- 결정:
  - 사용자가 `컴퓨터 유즈`, `컴퓨터`, `컴퓨터로 해`, `윈도우 조작`, `크롬에서 해`, `내가 띄워둔 크롬에서 해`라고 하면 `computer-use@openai-bundled` 의도로 해석한다.
  - 명시 플러그인 링크가 없어도 Windows/Chrome UI 조작 요청이면 `[컴퓨터사용]`으로 응답하고 Computer Use/Chrome 경로를 먼저 검토한다.
  - 운영 메모는 [[computer-use-codex]]에 누적한다.
