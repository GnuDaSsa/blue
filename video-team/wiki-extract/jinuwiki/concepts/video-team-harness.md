# 영상팀 하네스

사용자가 주는 기획서나 아이디어를 기반으로 음악, 이미지, 영상화를 순차 진행하는 표준 제작 하네스다.

## 핵심 원칙
- 모든 영상 업무는 [[video-workflow]]에 따라 위키를 먼저 읽는다.
- 각 응답은 짧은 한글 역할 태그로 시작한다: `[플래너]`, `[음악감독]`, `[이미지팀]`, `[영상감독]`, `[위키관리]`.
- 한 컷은 **최대 3초를 넘기지 않는다**.
- 시댄스/영상화 장면은 장면당 **최소 2.5초**를 가진다.
- 따라서 기본 컷 길이는 **2.5~3.0초** 범위로 잡는다.
- 음악의 스네어/클랩/강한 트랜지언트는 강하지 않더라도 장면 전환 기준으로 쓴다.
- 사용자 지적과 QC 기준은 [[image-qc-criteria]]에 누적한다.

## 0. 입력
사용자가 기획서, 아이디어, 레퍼런스, 키워드, 분위기 중 하나 이상을 제공한다.

필수로 파악할 것:
- 목적: 광고, 뮤직비디오, 숏폼, 아트필름, 제품컷, 내러티브 등
- 길이: 전체 목표 길이
- 톤: 감정, 색감, 속도, 장르
- 금지사항: 인물/얼굴, 브랜드, 텍스트, 로고, 민감 요소
- 산출물: 음악, 이미지, 영상, 프롬프트, 전체 패키지 중 무엇인지

## 1. [플래너] 기획 보강
플래너는 사용자의 원안을 먼저 보강한다.

출력:
- 한 줄 콘셉트
- 관객이 느껴야 할 감정
- 전체 구조
- 컷 수와 예상 길이
- 음악 방향
- 이미지 생성 방향
- 시댄스 영상화 방향
- 다음 단계 실행 계획

플래너는 이후 단계의 기준이 되는 `프로젝트 카드`를 만든다. 의미 있는 작업이면 `projects/`에 기록한다.

## 2. [음악감독] Suno 음악 후보 제작
음악감독은 `music-composition` 스킬을 활용해 Suno용 프롬프트를 만든다.

Suno 운영 규칙:
- 기본은 Custom Mode.
- 가사가 없는 음악은 Simple Mode에서 Instrumental을 체크하고 생성.
- 1회 Generate 시 2개의 곡이 나온다.
- 총 4회 Generate하여 **8개의 후보군**을 만든다.
- 4개의 Generate 프롬프트는 서로 조금씩 달라야 한다.

다양성 축:
- 템포: 예) 92 / 104 / 116 / 128 BPM
- 리듬 밀도: sparse / medium / energetic
- 질감: acoustic / synth / cinematic / band
- 감정: warm / tense / dreamy / playful
- 보컬 여부: vocal / instrumental
- 컷 전환 기준이 되는 스네어/클랩 존재감

후보 평가:
- 8개 후보를 듣고 3개 추천작을 고른다.
- 추천 이유는 영상 적합성 기준으로 설명한다.
- 사용자가 1개를 선택하면 그 곡을 기준으로 이후 작업을 시작한다.

음악 선택 후 추출할 것:
- BPM/체감 템포
- 에너지 곡선
- 장면 전환 후보 지점
- 스네어/클랩/강한 트랜지언트 위치
- 2.5~3.0초 컷 분할안

## 3. [이미지팀] GPT Image 2 이미지 제작
이미지팀은 선택된 곡과 컷 분할안을 기반으로 컷별 이미지를 만든다.

운영:
- Computer Use를 사용하지 않는다.
- Codex application runtime 또는 이미지 생성 백엔드에서 진행한다.
- 컷당 기본 1개 이상의 키프레임 이미지를 만든다.
- 각 컷은 최대 3초를 넘기지 않는 전제로 설계한다.
- 음악의 스네어/클랩/전환감을 이미지 콘셉트 변화 기준으로 삼는다.

이미지 프롬프트는 컷별로 다음을 포함한다:
- 컷 번호와 길이
- 장면 목적
- 피사체/배경
- 색감/조명
- 카메라 앵글과 렌즈감
- 움직임으로 확장될 여지
- 금지 요소

QC:
- [[image-qc-criteria]] 기준으로 자체 QC한다.
- 통과한 이미지만 영상화 단계로 보낸다.
- 사용자의 지적은 QC 기준에 누적한다.

## 4. [영상감독] 시댄스 영상화
영상감독은 `seedance2-skill`을 활용하고 Chrome 기준 브라우저 자동화로 시댄스/영상 생성 UI를 조작한다. Computer Use는 정상 연결될 때만 보조 수단으로 쓴다.

운영:
- 반드시 멀티 레퍼런스만 사용한다.
- 장면당 duration은 **최소 2.5초**.
- 한 컷은 **최대 3초**.
- 기본 장면 길이: 2.5~3.0초.
- 각 레퍼런스의 역할을 명시한다.
- 실제 사람 얼굴 업로드 제한을 먼저 확인한다.

멀티 레퍼런스 기본 구성:
- `@Image1`: 첫 프레임 또는 주 피사체/장면
- `@Image2`: 색감/무드/배경
- `@Audio1`: 선택된 Suno 음악 또는 리듬 기준
- 선택적으로 `@Image3`: 마지막 프레임 또는 세부 질감

프롬프트는 다음을 포함한다:
- `@` 레퍼런스 역할
- 장면 길이
- 카메라 움직임
- 피사체 움직임
- 음악/스네어 기준 전환
- 금지 요소

## 5. [위키관리] 기록
각 프로젝트는 위키에 누적한다.

기록 위치:
- `projects/`: 프로젝트 카드, 전체 결정, 최종 선택
- `prompts/`: Suno 프롬프트, 이미지 프롬프트, 시댄스 프롬프트
- `worklog/`: 작업 과정과 변경점
- `concepts/`: 반복되는 제작 규칙과 QC 기준

## 단계 요약
1. `[플래너]` 기획 보강 및 실행 계획
2. `[음악감독]` Suno 4회 Generate → 8곡 후보 → 3곡 추천 → 사용자 선택
3. `[이미지팀]` 선택곡 기반 컷 분할 → GPT Image 2 이미지 제작 → QC
4. `[영상감독]` 패스 이미지로 시댄스 멀티 레퍼런스 영상화
5. `[위키관리]` 결과와 기준 누적
